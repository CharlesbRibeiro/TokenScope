# TokenScope DD — Metodologia Simples de Conhecimento e Publicação

O Obsidian organiza o contexto, os aprendizados e o acompanhamento do TokenScope DD para que o fundador compreenda o projeto sem depender de linguagem técnica. O Vault oficial fica em `C:\Users\Usuario\Documents\RAW-OBISIDIAN` e permanece fora do repositório.

O repositório é a fonte oficial. O Vault apresenta essa verdade por meio de uma página principal, notas de produto, estado, tarefas, decisões, execuções, aprendizados, conceitos, perguntas e referências. As explicações começam em linguagem simples e, quando necessário, avançam por exemplo, aplicação no produto e detalhe técnico.

Depois de tarefas relevantes, o Codex registra a execução e atualiza apenas as notas afetadas por mudanças reais. Aprendizados podem originar ideias para LinkedIn ou outras publicações, mas não existe publicação automática: o fundador revisa e autoriza cada conteúdo.

Conteúdo privado permanece local, não é enviado a APIs e não deve ser copiado para o repositório sem aprovação. Nesta fase, o Vault não usa sincronização, conta conectada nem plugins externos.

Fluxo de trabalho:

`Fundador define e decide → ChatGPT organiza e audita → Codex executa, testa e registra → Obsidian explica e conecta → Git preserva somente o que foi aprovado`
