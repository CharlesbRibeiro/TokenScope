# Contribuindo

Leia `AGENTS.md` e a documentação obrigatória antes de alterar o projeto. Mantenha mudanças pequenas, locais e rastreáveis.

## Fluxo básico

1. Execute `git status` e preserve alterações existentes.
2. Consulte a base documental aplicável.
3. Implemente código reutilizável em `src/tokenscope_dd/` e testes em `tests/`.
4. Não inclua segredos, dados sensíveis ou artefatos pesados.
5. Execute as validações locais pertinentes.
6. Documente decisões arquiteturais em `docs/05_architecture_decisions.md`.

Commits, pushes e pull requests exigem solicitação explícita do responsável pelo projeto.
